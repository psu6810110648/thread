# Python Concurrency Simulation (Asyncio, Threading, Process Pool)

โปรแกรมนี้นำเสนอการจำลองการทำงานแบบคู่ขนาน (Concurrency) และแบบขนาน (Parallelism) ในภาษา Python เพื่อเปรียบเทียบและทำความเข้าใจการใช้งาน `asyncio`, `threading`, และ `process pool` ผ่าน 3 สถานการณ์จำลอง ตามโจทย์ Assignment

## ส่วนต่างๆ ของโปรแกรม

โปรแกรมนี้จำลอง 3 ขั้นตอนหลักของการทำงานที่เหมาะสมกับแต่ละเครื่องมือ ได้แก่

1. **การจำลอง Network I/O ด้วย `asyncio`**:
   - งานส่วนใหญ่มักจะหมดเวลาไปกับการรอเชื่อมต่อเครือข่าย โปรแกรมจำลองการดึงข้อมูลจาก API หลายๆ แหล่งพร้อมกัน `asyncio` เหมาะกับงานลักษณะ I/O-bound เช่นการรอ Request เนื่องจากตัวมันทำงานใน thread เดียวสลับกันไปมาเมื่อเกิดการรอ
   
2. **การจำลอง Disk I/O ด้วย `Threading`**:
   - นำข้อมูลทีได้จำลองการบันทึกลงไฟล์ (Disk) แม้ Python มี GIL (Global Interpreter Lock) แต่ Thread จะมีประโยชน์เวลาเกิดคิว I/O Blocking เช่นการประมวลผลไฟล์หรือรอฐานข้อมูล
   
3. **การประมวลผล CPU-bound ด้วย `Process Pool` (`multiprocessing.Pool`)**:
   - นำตัวเลขขนาดใหญ่มาคำนวณหาตัวประกอบเฉพาะ (Prime factorization) ซึ่งเป็นงานที่ต้องใช้ CPU ทำงานล้วนๆ งานลักษณะนี้ `Process Pool` จะสร้าง Process โดยอิสระทำให้สามารถแยกโหลดไปรันบนแต่ละ CPU Core ได้อย่างมีประสิทธิภาพ ทำงานขนานกันแบบข้ามขีดจำกัด GIL
